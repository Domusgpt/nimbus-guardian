Claude Code

Meet the Observability Plugin

Drop slash commands straight into Claude Code. The plugin mirrors the dashboard telemetry, so every conversation can surface status, streaming, history, and incident context instantly.

Included Commands

  • /observability-status – live snapshot with sink health.
  • /observability-history – rolling window with summary insights.
  • /observability-stream – SSE stream viewer with graceful teardown.
  • /observability-incidents – filterable incident tracker.
  • /observability-digest – snapshot, history, and incident bundle.

Trust & Install

  1. Open Claude Code in the repository root.
  2. Run /plugin marketplace add ./claude-plugins.
  3. Install /plugin install nimbus-guardian-observability@claude-plugins.
  4. Type /observability-status to confirm the commands are active.

All command docs live inside the plugin for quick refreshers.

CLI Digest

One Command for Total Telemetry

Ship digest reports from your terminal or automation. Batching, JSON output, and headers are all baked in so the CLI stays in sync with Claude Code and the dashboard.

Quick Start

npm install -g nimbus-guardian
nimbus observability-digest --history --incidents --format table

Use --format json for automation pipelines or --once for a single-shot stream.

Outputs You Get

  • Latest snapshot with fingerprint and sink summaries.
  • Rolling history including overview analytics.
  • Incident ledger with duration, status, and reason codes.
  • Exit codes wired for CI/CD fast-fails.
Orbit Terminal

Preview the Commands Before You Install

Spin through live syntax, clipboard-ready env vars, and curated tips for every observability surface. The orbit terminal mirrors the Claude plugin docs so operators always know which command answers which question. 

nimbus observability-status \
  --timeout 8000 \
  --format table
nimbus observability-stream \
  --cookie "guardian_session=..." \
  --heartbeat 15000
nimbus observability-history \
  --limit 10 \
  --overview
nimbus observability-incidents \
  --status degraded,failing \
  --format json
nimbus observability-digest \
  --history --incidents \
  --output ./orbit-digest.json

Why Run Status?

Get a one-shot snapshot with fingerprint deltas, sink summaries, and health badges. Perfect for quick diagnostics inside CI or Claude conversations.

Why Run Stream?

Watch fingerprints arrive in real time. Graceful teardown keeps terminals tidy while sink metrics update the moment exporters emit fresh telemetry.

Why Run History?

Review the rolling buffer, summary analytics, and incident counts in one go. Handy for daily standups or retro decks.

Why Run Incidents?

Filter for degraded or failing windows to see how long each sink struggled and what triggered the alert.

Why Run Digest?

Bundle a snapshot, history insights, and the incident ledger into one JSON or table-friendly report. Operators ship it to Parserstor partners in seconds.
